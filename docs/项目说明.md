# 项目管理系统 · 项目说明

一个面向团队的 **需求 / 任务 / Bug 协作平台**，前后端分离，企业微信群艾特通知，Docker 一键部署。

- **源码**：https://github.com/leeazhuang/project-mgmt
- **镜像**：`registry.cn-hangzhou.aliyuncs.com/leehaha-public/project-mgmt:latest`
- **部署**：见 [镜像部署说明.md](镜像部署说明.md)
- **操作手册**：见 `操作手册/`（业务人员使用指南，含截图）

---

## 一、技术栈

| 层 | 技术 |
|----|------|
| 前端 | Vue 3 + Vite + Element Plus + Pinia + ECharts + wangEditor |
| 后端 | Python 3.9 + FastAPI + SQLAlchemy 2.0 + Uvicorn |
| 数据库 | MySQL（PyMySQL 驱动）|
| 鉴权 | JWT（python-jose）+ bcrypt 密码哈希 |
| 文件存储 | 本地磁盘 / 阿里云 OSS（可切换）|
| 通知 | 企业微信 HOOK 机器人（群艾特）|
| 部署 | Docker 单镜像（Nginx + Uvicorn + Supervisor + 定时任务）|

---

## 二、核心功能

| 模块 | 说明 |
|------|------|
| **项目管理** | 项目创建、成员管理，每个项目设「项目负责人」「技术负责人」|
| **需求管理** | 草稿 → 提交 → 审批 → 排期 → 开发 全流程审批流 |
| **任务管理** | 多人指派、个人状态流转、任务看板、我的任务、延期/作废 |
| **Bug 管理** | 提交 → 指派 → 修复 → 验证 → 重开 流转，Bug列表 / 我的Bug |
| **消息通知** | 所有提醒通过企业微信群**艾特**对应负责人，含项目名前缀 |
| **系统管理** | 用户（绑定企微群成员）、角色、菜单、系统设置、操作日志 |

## 三、角色与权限体系

### 两套权限并行

1. **系统角色**（决定能看到哪些菜单）：
   超级管理员、项目负责人、技术负责人、开发人员、产品、Bug反馈
2. **项目内身份**（决定在具体项目里能做什么）：
   每个项目的 `owner_id`(项目负责人) 和 `tech_leader_id`(技术负责人)

> 多数写操作（审批需求、创建任务、指派 Bug）看的是"你是不是这个项目的负责人/技术负责人"，而非系统角色。

### 数据权限（后端 `services/data_permission.py`）

- 非超管只能访问自己**所在项目**的数据
- 开发人员的任务列表/看板/我的任务**只看分配给自己的**
- 详情/列表/日志/评论/附件均按项目成员校验，越权返回 403（前端自动跳首页）

## 四、关键设计

### 1. 启动自动初始化（`app/db_init.py` + `app/seed_data.sql`）
容器启动时：数据库不存在 → 自动建库 → 建所有表 → 空库则导入种子数据（超管 admin、6 角色、菜单、权限、config 空值）。别人部署只需填数据库连接，开箱即用。
> 种子数据由 `gen_seed.py` 从真实库导出生成，改菜单/角色后重跑即可刷新。

### 2. 企业微信群艾特通知（`services/notify_service.py` + `wx_bot_service.py`）
- 通过企微 HOOK 机器人向用户绑定的群发**艾特消息**
- 用户在「用户管理」绑定企微群 + 群成员
- 通知**挂在数据库事务上**，commit 成功才真正发送，回滚则丢弃（保证"业务生效才通知"）
- 机器人发送失败 → 兜底通过企微 Webhook 告警并艾特运维

### 3. 附件存储双模式（`services/oss_service.py`）
- 默认存本地 `uploads/`，开启 OSS 后存阿里云（可配存储路径前缀）
- 系统设置界面切换，富文本图片同样支持

### 4. 操作日志（`middleware/operation_log.py`）
全局中间件自动记录所有写操作（含请求参数，密码类字段脱敏），无需逐接口埋点。

## 六、本地开发

```bash
# 后端
cd backend
pip install -r requirements.txt
# 准备 .env（DB 连接），然后：
uvicorn app.main:app --reload --port 8000

# 前端
cd frontend
npm install
npm run dev          # 默认 5173，已配 /api 代理到 8000
```

启动后端会自动建库初始化，默认管理员 **admin / 123123**。

## 八、注意事项（开发踩坑记录）

- **Python 3.9**：不支持 `X | None` 联合类型语法，用 `Optional`
- **bcrypt 版本**：必须 `bcrypt==4.0.1`（4.1+ 与 passlib 1.7.4 不兼容，登录会报 "password cannot be longer than 72 bytes"）
- **Docker 构建**：docker hub 国内常拉不动，Dockerfile 已改用阿里云/国内镜像源
- **容器连数据库**：`DB_HOST` 不能用 localhost（指容器自己），要填实际 IP
